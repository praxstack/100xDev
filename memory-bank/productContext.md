# Product Context — Top-Down Learning System for Prax

## Problem Statement

Prax is a skilled software engineer (3+ years at Amazon) transitioning to a focused learning phase. Despite strong practical experience building production systems, there are significant gaps in:
- DSA foundations (forgot DFS, BFS, DP, Graphs, Heaps)
- Java Advanced (Multithreading, Concurrency, Collections internals)
- System Design (never formally studied)
- Design Principles (SOLID, patterns — doesn't know when/where to apply)

The core problem: **Knowing how to build doesn't mean understanding why it works.**

## Value Proposition

Gabriel Petersson's methodology provides:
1. **Top-Down Learning**: Start with the problem, learn theory when needed
2. **Recursive Gap-Filling**: Drill down until genuine understanding ("click")
3. **No Vibe Coding**: Understand foundations of every shortcut
4. **Teach-Back Verification**: Prove understanding by explaining
